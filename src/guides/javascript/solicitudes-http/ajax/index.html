¿Qué es una petición HTTP?

Una petición o solicitud HTTP es lo que suele denominarse a la acción por parte de un navegador de solicitar a un servidor web un recurso ya sea un documento o archivo, como archivos .html, .js, una imagen, una fuente tipográfica, etc. Por medio de esta petición, el navegador puede descargar ese archivo, alcamenacenarlo en una caché temporal de archivos del navegador y, finalmente, utilizarlo para mostrarlo en la página actual que lo ha solicitado.

Peticiones HTTP mediante AJAX

Con el pasar del tiempo y las constantes mejoras de rendimientos de la web, aparece una nueva modalidad para realizar peticiones, denominada AJAX (Asynchronous javascript and XML). Esta modalidad se basa en que la petición HTTP se realiza desde javascript, de forma transparente al usuario, descargando la información y pudiendo tratarla sin necesidad de mostrarla directamente en la página.

Esto produce un interesante cambio en el panorama que había hasta entonces, puesto que ahora podemos hacer actualizaciones de contenidos de forma parcial, de modo que se actualice una página en <<vivo>>, sin necesidad de recargar toda la página, sino solamente actualizando una pequeña parte de ella, pudiendo utilizar javascript para crear todo tipo de lógica de apoyo.

Métodos de petición AJAX

Existen varias formas de realizar peticiones HTTP mediante AJAX, pero las principales suelen ser XMLHttpRequest y fetch (nativas, incluidas en el navegador por defecto), además de librerías como axios o superagent:

Método Descripción
XMLHttpRequest Abreviado como XHR. El más antiguo. Nativo
fetch Nuevo sistema nativo de peticiones basado en promesas. Nativo
Axios Librería basada en promesas para realizar peticiones en Node o en navegadores
superagent Librería para realizar peticiones HTTP tanto en Node como en navegadores

MPA: Multiple Page Application

Tradicionalmente, el sistema que se seguía para crear páginas o aplicaciones web se enmarcaba dentro de la categoría de páginas MPA (Multiple Page Application). Bajo este sistema, el navegador se descarga el archivo .html, lo lee y luego realiza las peticiones de los archivos restantes relacionados que encuentra en el documento HTML como archivos .css, .js, etc. Si el usuario pulsa en algún enlace, se descarga el .html de dicho enlace (recargando la página completa) y se repite el proceso.
